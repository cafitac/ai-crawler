# AI Harness Contract

`ai-crawler compile` is the URL-based one-command entrypoint for agent/harness use. `ai-crawler auto` is the evidence-based entrypoint when a harness already has an evidence file.

## Commands

URL-based one-command mode:

```bash
ai-crawler compile https://example.com/products --goal "collect products" --json
```

Evidence-based mode:

```bash
ai-crawler auto evidence.json --json
```

## Local verification

```bash
bash scripts/verify-ai-harness.sh
```

Default artifacts:

- `evidence.json`: browser probe evidence, only generated by `compile` or `probe`
- `recipe.yaml`: initial deterministic baseline recipe
- `repaired.recipe.yaml`: repaired recipe after test diagnostics
- `test.jsonl`: initial test crawl output
- `crawl.jsonl`: final crawl output
- `auto.report.json`: same payload printed to stdout in `--json` mode

## Exit codes

- `0`: final recipe test completed and extracted items successfully.
- `2`: probe/generation/test completed far enough to produce a machine-readable failure report, but final classification is not `success` or no endpoint candidates were found.

Even on exit code `2`, `--json` mode prints the report payload to stdout and writes `auto.report.json` so the caller can inspect the failure category and `failure_phase`.

## Stdout/stderr policy

- In `--json` mode, stdout contains exactly one compact JSON object.
- Human-readable diagnostics, including probe/generation failures, go to stderr.
- Sensitive values in samples and diagnostics must be redacted before stdout/report writes.

## Failure categories

- `success`: final request completed and items were extracted.
- `no_endpoint_candidates`: `compile`/`probe` did not capture useful replayable API candidates. Retry probe on an authorized target, inspect `evidence.json`, or use split mode to provide evidence manually.
- `probe_failed`: browser probe setup or navigation failed. Check browser extras/Playwright installation, target reachability, and authorization.
- `extraction_failed`: HTTP response succeeded but recipe extracted no items.
- `http_error`: non-2xx response without a detected challenge boundary.
- `no_response`: no test response was captured.
- `challenge_detected`: challenge/login/bot-boundary style response detected. The crawler does not bypass this; callers should use manual handoff or an authorized session path.

## Failure phase triage

- `probe`: browser discovery failed before evidence could be written.
- `generate`: evidence was written, but no useful endpoint candidate could be selected.
- `final_test`: recipe generation/repair ran, but the final deterministic HTTP replay did not classify as `success`.
- empty string: success.

## Redaction boundary

`body_sample` is capped and passed through common secret redaction before it is written to reports or stdout. Authorization bearer values, cookie session/token values, and common token/API-key assignments are replaced with `[REDACTED]`.
